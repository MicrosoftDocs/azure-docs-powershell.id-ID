# Invoke-AzStorageSyncFileRecall

## SYNOPSIS
Perintah ini mengembalikan semua file tingkat ke disk lokal.

## SINTAKS

### StringParameterSet (Default)
```
Invoke-AzStorageSyncFileRecall [-ResourceGroupName] <String> [-StorageSyncServiceName] <String>
 [-SyncGroupName] <String> [-Name] <String> [-Pattern <String>] [-RecallPath <String>] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdParameterSet
```
Invoke-AzStorageSyncFileRecall [-ResourceId] <String> [-Pattern <String>] [-RecallPath <String>] [-PassThru]
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ObjectParameterSet
```
Invoke-AzStorageSyncFileRecall [-InputObject] <PSServerEndpoint> [-Pattern <String>] [-RecallPath <String>]
 [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESKRIPSI
Ketika tingkatan awan diaktifkan pada titik akhir server (lokasi tertentu pada server terdaftar), maka perintah ini dapat digunakan untuk memanggil kembali semua file tingkat ke disk lokal. Sangat disarankan untuk menonaktifkan tingkatan cloud pada titik akhir server ini sebelum memulai penarikan kembali. Jika tingkatan masih ada, penarikan mungkin mengarah pada file lain yang terhingkup yang melewatkan tujuan yang diinginkan dari semua konten yang berada di disk lokal.

## CONTOH

### Contoh 1
```powershell
PS C:\> Invoke-AzStorageSyncFileRecall -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -ServerEndpointName "myServerEndpointName"
```

Perintah ini secara rekursif menarik kembali semua file berjenjang yang berada di bawah jalur akar dari titik akhir server yang ditentukan.
